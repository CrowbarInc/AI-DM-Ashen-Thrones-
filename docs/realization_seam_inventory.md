# Realization Seam Inventory

Branch context: `feature/failure-locality`

Purpose: concrete inventory of Narrative Realization / Prompt Realization seams between:

`Planner > GPT / Prompt Realization > Gate`

Principle used for classification: Planner decides what matters and what obligations exist. GPT / Prompt Realization may only realize authorized narration in words. It must not reconstruct truth, invent semantics, or create hidden fallback prose.

Status terms:

- `SAFE`: packaging/validation only, or prose is constrained to an explicitly authorized upstream artifact.
- `BOUNDED`: may emit or mutate prose, but from narrow, inspectable contracts or visible/authorized inputs.
- `LEGACY`: pre-planner or compatibility path still present.
- `SUSPICIOUS`: can synthesize player-facing prose or read raw state in a way that should be hardened.
- `UNKNOWN`: ambiguous from static inspection; do not assume safe.

## Highest-risk seams

- `game.gm.call_gpt` returns a hard-coded player-facing API failure line on exception before the API layer's fast-fallback path can own all emergency prose.
- `game.api._build_gpt_narration_from_authoritative_state` still has multiple emergency exits (`planner_convergence`, upstream API fast fallback, forced retry fallback, manual GPT budget) that can bypass normal plan-driven realization.
- `game.gm.build_messages` can still append `resolution["hint"]` into prompt instructions outside action-outcome mode.
- `game.gm.apply_response_policy_enforcement` contains deterministic post-GPT prose rewrites that read scene/session/world/resolution after prompt construction.
- `game.gm_retry.apply_deterministic_retry_fallback` can replace GPT text with known-fact, social, open-crowd, or uncertainty fallback prose.
- `game.gm_retry.force_terminal_retry_fallback` and minimal social/nonsocial repair helpers are terminal prose writers after GPT failure.
- `game.final_emission_gate.apply_final_emission_gate` has many text replacement layers and sealed fallback branches, including opening, social, anti-reset, global scene, and acceptance-quality paths.
- `game.final_emission_gate._deterministic_opening_fallback_text_and_meta` composes opening fallback prose at the gate from curated facts.
- `game.upstream_response_repairs.build_upstream_prepared_emission_payload` intentionally creates contract-shaped fallback prose upstream; safe only if treated as the sole authorized fallback-prose source for the gate.
- `game.diegetic_fallback_narration` is a library of fallback prose renderers from visible facts / scene anchors; every caller must prove authorization and locality.

## Notes

- This is a static inventory, not a behavior change.
- `prompt_context` is mostly a renderer/packager, but still reads large raw state slices. Those reads are marked `Yes` under "Reads raw state after Planner" even when current code appears bounded.
- Gate-adjacent paths are not automatically unsafe. They become high risk when they compose new player-facing text from raw state, prior model output, or generic fallback templates instead of selecting planner/upstream-prepared prose.
- `UNKNOWN` means the seam needs targeted follow-up inspection; it does not imply safe or unsafe.
